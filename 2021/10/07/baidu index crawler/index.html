Baidu Index Crawler

How to use

  • change .py path in start

  • check for configurations in configs

    • mode-which mode you want, like work/test/history

    • db-which db to save

    • spider-which spider to use

    • scheduler-specific scheduler with spider

      • task preparation and filter

  • execute start.py

Features

  • word spider: no word spider but just area 0 and year all tasks

    • reliable word check relies on a new api
      https://index.baidu.com/api/AddWordApi/checkWordsExists?word={word}
    • api http://index.baidu.com/api/SugApi/sug?inputword[]={word} only for suggesting word check, not reliable
      for word like 万科a which is included but not showed in this api’s response

  • exporter for data: moved to jupyter notebook for interactive dealing

Change Log

  • supporting multi-type tasks

    • first level word check and general crawling: from very start, when checking whether a word included or not;
      what’s more, it can show which years it has completely null search from national level, and this
      can reduce a lot of work
      • eg. for word 华熙生物, which has no search till 2020 year at national level, thus we can exclude
        work related to this word before 2020 year and all areas
    • second level crawling for yearly search index and provincial level, based on first level result and focus
      mainly on those years with search index
    • third level crawling for yearly search index on city level
